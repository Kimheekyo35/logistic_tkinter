# 🚀 물류팀의 송장번호 수집 및 FM 라벨 출력 업무를 자동화하는 Python 기반 Tkinter 응용프로그램 개발

## 프로젝트 전체 흐름
`로그인(Tkinter) → 쿠키 저장(runtime/states/*.json) → 작업 선택(FM 라벨 / 송장번호) → 페이지 접근 → 처리 및 PDF 저장 → 로그 출력`
## 프로젝트 개요
물류팀의 송장번호 처리 및 FM 라벨 출력 업무를 자동화하기 위해 Python 기반 프로그램을 개발했습니다.
초기에는 Streamlit 서버 배포 방식으로 구현했으나, 실제 운영 환경에서 오류 추적, 쿠키 갱신, 저장 경로 등 여러 문제가 발생해 최종적으로 Tkinter 기반 데스크톱 응용프로그램으로 재구성했습니다.

### 1. 주요 역할

- 송장번호 처리 / FM 라벨 출력 업무 프로세스 분석
- Shopee 로그인 구조 분석 및 쿠키 재사용 방식 설계
- Tkinter 기반 GUI 응용프로그램 개발
- 로그 출력, 오류 확인, PDF 저장 확인 기능 추가
- 로그인 세션은 runtime/states 디렉토리에 JSON 파일로 저장하고 재사용하도록 구성
- JavaScript를 활용해 PDF를 직접 다운로드하고, 결과 파일은 {Brand}_{YYYY}_{MM}_{DD} 규칙으로 저장
- 송장번호 결과물은 대량 데이터를 고려해 1200개 단위로 분할 저장
- GUI 상에서 로그를 출력해 오류 발생 지점과 PDF 저장 여부를 확인 가능하도록 구성
- 비개발자 사용 환경을 고려한 배포 방식 개선
  
### 2. 문제 상황 및 해결 방법

```
1️⃣ Headless 환경에서는 오류 위치를 파악하기 어려움 + Shopee 특성상 예외 상황이 잦아 운영 안정성이 낮음
   ➡️ GUI 상에서 로그와 상태를 확인할 수 있도록 구성하여 오류 추적 가능하게 개선

2️⃣ 새 페이지 진입 시 반복 로그인이 필요해 실행이 번거로움
   ➡️ 로그인 단계에서 쿠키를 JSON으로 저장하고 이후 작업에서 재사용

3️⃣ 쿠키를 2~3일마다 갱신해야 하나 Streamlit 재배포 방식은 유지보수 부담이 큼
   ➡️ Streamlit 서버 배포 대신 Tkinter 기반 로컬 실행 구조로 전환

4️⃣ 자회사/본사 간 공유폴더 접근 제약으로 저장 구조가 비효율적이었음
   ➡️ tkinter를 이용해 사용자의 PC에 직접 파일이 저장되도록 개선
```

### 🥇 성과
- 비개발자도 터미널 없이 실행 가능한 형태로 전달
- 오류 발생 위치와 PDF 저장 여부를 즉시 확인 가능
- 운영 편의성과 유지보수성 향상
- 물류팀 반복 업무 시간 절감
- **⭐7시 이전 퇴근 가능, 약 2시간의 여유시간 확보에 기여⭐**
